The Root Sequence (Ordered List)

  1. GRUB Bootloader Execution - The system loads the Gran Unified Bootloader from the EFI system partition, allowing selection of kernel parameters.
  2. Kernel Initialization - The Linux kernel decompresses into memory, initializes core hardware drivers, and mounts the temporary root filesystem (initramfs).
  3. Systemd Initialization - The kernel hands off control to PID 1, which begins orchestrating user-space services and target transitions.
    • Mounting essential virtual filesystems including /proc, /sys, and /dev for devide node communication.
    • Executing low-level early boot targets like sysinit.target to prepare basic system initialization.
    • Activating multi-user.target to bring up the standard non-graphical multi-user server environment.
      1. Allocating the sshd.service daemon to allow secure remote administrative access.
      2. Validating NetworkManager.service to establish local network interfaces and IP allocation.
      3. Initializing the postgresql.service socket to prepare the database engine for application connections.
