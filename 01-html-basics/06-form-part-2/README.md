# HTML Forms Practice - Part 2: Advanced Inputs

A semantic HTML exercise expanding on basic form structures. This project focuses on implementing multi-choice fields, mutually exclusive options, dropdown selections, and form routing parameters.

## 🎯 Objectives & Learnings

This second part of the form challenge introduces more complex data-gathering elements:

* **Dropdown Selection Menus (`<select>` & `<option>`):** Built structured selection dropdowns for data collection, including setting standard back-end `value` attributes and forcing pre-selected entries via the `selected` attribute.
* **Selection Checkboxes (`type="checkbox"`):** Implemented multi-selection groupings for overlapping data points (such as dietary needs and tracking preferences) using the `checked` state property for standard settings.
* **Exclusive Radio Groups (`type="radio"`):** Created strict single-choice pools (e.g., shirt sizing and ticket levels) by grouping distinct input tags together under identical `name` attributes.
* **Form Action Configuration:** Applied `action="confirmation.html"` and `method="post"` parameters directly to the root `<form>` component to route mock data delivery paths locally.
* **Input-to-Label Hookups:** Reinforced web standards by explicitly mapping distinct `<label for="...">` tags to their absolute matching field `id` properties to expand interactive click areas.
